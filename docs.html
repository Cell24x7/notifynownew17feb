Overview

NotifyNow is an enterprise-grade multi-channel marketing API. It allows users to send promotional and transactional messages across WhatsApp, RCS, SMS, and Voice seamlessly. The API provides robust mechanisms for managing contacts, scheduling campaigns, integrating webhooks, and tracking real-time delivery reports (DLR).

Base URL: https://api.notifynow.in/api/v1

✨ Release Notes (Latest Updates)

Version 2.4.0 — High-Volume Scalability & Link Tracking Update

June 18, 2026

🚀 Database Normalization & Panel Speed

  • Auto-Archival System: `message_logs_archive` & `api_message_logs_archive` created. Background cron automatically moves records older than 90 days out of active tables, ensuring panel reports load instantly regardless of campaign sizes.
  • Queue Indexing: Critical composite indices (status, campaign_id) added to `campaign_queue` and `api_campaign_queue`. Eliminates "Row Locking" during multi-million sending.
  • Report Acceleration: Composite indices `idx_user_camp_status` added to message tables for lightning-fast delivery/read statistics generation.
  • Daily Stats Engine: New `campaign_daily_stats` table introduced to prevent re-counting millions of rows for dashboard graphs.

🔗 Short Link Click Tracking

  • Campaign Click Tracking: The sending engine now automatically replaces URLs inside your templates with trackable Short Links (e.g., `s.notifynow.in/xyz123`).
  • Real-time Analytics: When a user clicks the link, the click is logged against their specific `message_id` and `campaign_id` in the `short_link_clicks` table.
  • Automatic Redirection: Clicks are processed instantly and redirected to the original long URL without delay.

🛠️ Developer APIs & Core Updates

  • Contact Tags are now fully indexed on the `contact_tags` junction table (idx_user_tag), making segment querying 10x faster.
  • Deployment script updated to run schema migrations sequentially to prevent MySQL out-of-memory errors on new servers.

Architecture Updates (DB Normalization)

Scalability & Normalization Plan Implemented

To ensure zero downtime and lightning-fast panel reports during massive multi-million campaigns, the database schema and background queues have been deeply optimized:

  • Archival Strategy: Logs older than 90 days are now seamlessly offloaded to message_logs_archive and api_message_logs_archive tables. This keeps active tables lightweight and prevents panel timeout issues.
  • Queue Optimization: `campaign_queue` and `api_campaign_queue` now leverage composite indices (status, campaign_id), eliminating table-locks during active background polling. Native integration with Redis / BullMQ handles ultra-high concurrency safely.
  • Report Acceleration: Real-time indexing (idx_user_camp_status) added to all logging tables. Added `campaign_daily_stats` summary table for instant analytics rendering without counting millions of rows.
  • Contact Tagging Normalization: Labels are now fully indexed on the `contact_tags` junction table (idx_user_tag), drastically speeding up segment targeting.

Authentication

NotifyNow uses two forms of authentication depending on your use case:

  • JWT Bearer Token: For panel operations (Contacts, Campaigns, Reports). Provided in the Authorization: Bearer <token> header.
  • API Key (API Password): For server-to-server transactional messaging via Webhooks or standard API integration. Passed in the payload or headers depending on the route.
POST /auth/login
Panel Login

Authenticate and retrieve your JWT access token.

FieldTypeRequiredDescription
emailstringYesRegistered email
passwordstringYesAccount password
curl -X POST https://api.notifynow.in/api/v1/auth/login \ -H "Content-Type: application/json" \ -d '{"email": "user@example.com", "password": "password123"}' // Response { "success": true, "token": "eyJhbGciOiJIUz...", "user": { "id": 1, "role": "user" } }
